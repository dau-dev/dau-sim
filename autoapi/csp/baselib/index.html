csp.baselib.log(level: int, tag: str, x, logger: logging.Logger | None = None, logger_tz: object = None, use_thread: bool = False)[source]¶

Logs a time-series value during the execution of a csp graph

Arguments: level: log level to use e.g. CRITICAL tag: label for the logged entry e.g. “x” x: the time-series to log logger: optional, the logger to use. If not set, the default csp logger is used logger_tz: optional, time-zone that the logger is set to. use_thread: whether or not to run the log node in a separate thread

  • This feature is useful for expensive log calls that will be printing large objects, such as structs

  • NOTE: If use_thread=True and a time-series value is modified after log is called, it may log the modified version.

In this case, which is rare, you must pass a copy of the time-series to the log node to ensure correct behavior.

csp.baselib.exprtk(expression_str: str, inputs: Dict[str, csp.impl.types.tstype.ts[object]], state_vars: dict = {}, trigger: csp.impl.types.tstype.ts[object] = None, functions: dict = {}, constants: dict = {}, output_ndarray: bool = False) → csp.impl.types.tstype.ts[float | numpy.ndarray][source]¶
given a mathematical expression,

and a set of timeseries corresponding to variables in that expression, tick out the result (a float) of that expression, either every time an input ticks, or on the trigger if provided.

Parameters:

  • expression_str – a mathematical expression as per the C++ Mathematical Expression Toolkit Library, ExprTk: http://www.partow.net/programming/exprtk/

  • inputs – a dict basket of timeseries. The keys will correspond to the variables in the expression. The timeseries can be of float or string

  • state_vars – an optional dictionary of variables to be held in state between executions, and assignable within the expression. Keys are the variable names and values are the starting values

  • trigger – an optional trigger for when to calculate. If not provided, will calculate any time an input ticks

  • functions – an optional dictionary whose keys are function names that can be used in the expression, and whose values are of the form ((“arg1”, ..), “function body”), for example {“foo”: ((“x”,”y”), “x*y”)}

  • constants – an optional dictionary of constants. Keys are names and values are the constant values

  • output_ndarray – flag, defaults to False, if True, will output ndarray instead of float. Must have expression use return like “return [a, b, c]”

csp.baselib.wrap_feedback(i: csp.impl.types.tstype.ts[T]) → csp.impl.types.tstype.ts[T][source]¶

A convenience function to wrap the given time series as a feedback. Useful when using in conjunction with some type of delayed nodes. Usually the example would be something like this:

# The assumption here is that fill generates somehow fills for all orders. # So there exists a loop of strategy -> orders -> fills -> strategy, fills must be implemented as some kind of delayed node # that is instantiated at late stage after all “ordering” nodes have been created. This must create a feeback loop and as convinience # we mark the fills as a feedback by using a call to wrap_feedback. my_fills = get_fills_ts(…) my_orders = my_strategy(wrap_feedback(my_fills)) publish_orders(my_orders)

Parameters:

i – The input node to be wrapped as feedback

Returns:
