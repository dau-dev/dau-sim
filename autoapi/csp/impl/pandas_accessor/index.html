synchronize(threshold: datetime.timedelta)[source]¶

Synchronizes the series to “bucket” ticks together. When one element ticks, will wait until all other elements in the series tick before returning, or until the threshold has passed. This is particularly useful when doing cross-sectional analysis on data that is generated syncrhonously, but published asynchronously by symbol (i.e. bars).

Run a graph containing the edges in the series.

Parameters:

  • endtime – The start time for the graph

  • realtime – Whether to use the realtime csp engine

  • tick_count – The maximum number of ticks to return. Default is -1 (all ticks)

run(starttime: datetime.datetime, endtime: datetime.datetime, realtime: bool = False, tick_count: int = -1, tick_history: datetime.timedelta = timedelta(), snap: bool = False, collect: bool = True)[source]¶

“Run” a graph containing the edges in all the TsDtype series.

Parameters:

  • starttime – The start time for the graph

  • endtime – The start time for the graph

  • realtime – Whether to use the realtime csp engine

  • tick_count – The maximum number of ticks to return. Default is -1 (all ticks)

  • tick_history – The maximum tick history to return. Default is timedelta() (all ticks)

  • snap – Whether to run in “snap” mode, i.e. return the first tick of every series and stop once all series have ticked

  • collect – Whether to collect all the data in a single series before publishing. Improves the performance for publishing asynchronous time series, and can handle the case of duplicate timestamps. However, it is slower for synchronous time series.

Returns:

A new DataFrame with an additional index level containing the datetimes of the ticked values. Columns of type TsDtype are replaced by their ticked values. Values from static columns (not of TsDtype) are repeated for each timestamp.

snap(timeout: datetime.timedelta = timedelta(seconds=10), starttime: datetime.datetime = None)[source]¶

“Snap” a graph containing the edges in the series.

Parameters:

  • timeout – How long to wait for all the elements to tick before stopping the circuit

  • starttime – The start-time for the snap. Could be a time in the past (for snapping historical data) or a time in the future (to schedule a snap at, i.e. the next minute)

Returns:

A new DataFrame with the same indices where every Edge is replaced by it’s snapped value.
