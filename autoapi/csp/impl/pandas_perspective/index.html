class csp.impl.pandas_perspective.CspPerspectiveTable(data: pandas.DataFrame, index_col: str = 'index', time_col: str | None = 'timestamp', throttle: datetime.timedelta | None = timedelta(seconds=0.5), keep_history: bool = True, limit: int = None, localize: bool = False)¶
Parameters:

  • data – A data frame containing regular columns as well as TsDtype columns

  • index_col – The name of the column in perspective which will hold the index of the data frame. If the index of the data frame already has a name, this name will be used instead.

  • time_col – The name of the column to hold timestamps. If not provided, no timestamp column will be added.

  • throttle – The rate at which we update the perspective table from the buffer

  • keep_history – Whether to keep the history of all updates to the table. If false, will update in place. time_col must be supplied if keep_history is True.

  • limit – The maximum number of (most recent) rows the perspective table will hold. Only works when keep_history=True.

  • localize – Whether to localize time_col to local timestamps in the perspective view. If False, will display utc time.

Runs the dataframe explicitly upfront, then creates a perspective table out of it in one go. This can be significantly faster than incrementally updating the table from a historical graph.

Start a graph that sends data to the table on a csp engine thread. If clear=True, will clear any data from the data before writing to it. If auto_shutdown=True, will stop the engine if the table gets garbage collected.
