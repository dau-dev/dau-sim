:param broker - broker URL :param start_offset - signify where to start the stream playback from (defaults to KafkaStartOffset.LATEST ). Can be

one of the KafkaStartOffset enum types. datetime - to replay from the given absolute time timedelta - this will be taken as an absolute offset from starttime to playback from

:param group_id - ( optional ) if set, this adapter will behave as a consume-once consumer. start_offset may not be

set in this case since adapter will always replay from the last consumed offset.

subscribe(ts_type: type, msg_mapper: csp.adapters.utils.MsgMapper, topic, key=None, field_map: dict | str = None, meta_field_map: dict = None, push_mode: csp.PushMode = csp.PushMode.LAST_VALUE, adjust_out_of_order_time: bool = False, tick_timestamp_from_field: str = None, include_msg_before_start_time: bool = True)[source]¶

Subscribe to a Kafka topic and map incoming messages to CSP timeseries.

Parameters:

  • ts_type – The timeseries type to map the data to. Can be a csp.Struct or basic timeseries type

  • msg_mapper – MsgMapper object to manage mapping message protocol to struct

  • topic – Topic to subscribe to

  • key – Key to subscribe to. If None, subscribes to all messages on the topic. Note: In this “wildcard” mode, all messages will be marked as “live”

  • field_map – Dictionary of {message_field: struct_field} mapping or string for single field mapping

  • meta_field_map – Dictionary mapping kafka metadata to struct fields. Supported fields: - “partition”: Kafka partition number - “offset”: Message offset - “live”: Whether message is live or replay - “timestamp”: Kafka message timestamp - “key”: Message key

  • push_mode – Mode for handling incoming messages (LAST_VALUE, NON_COLLAPSING, BURST)

  • adjust_out_of_order_time – Allow out-of-order messages by forcing time to max(time, prev_time), only applies during sim replay.

  • tick_timestamp_from_field – Override engine tick time using this struct field. Only applies during sim replay

  • include_msg_before_start_time – Include messages from Kafka with times (either the time from Kafka or from the message as specified with tick_timestamp_from_field) before the engine start time.
